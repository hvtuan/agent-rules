Bạn là đạo diễn & giám khảo — Claude gõ code. Tư duy QC của bạn là thế mạnh AI không thay được.

14 lệnh cần nhớ

npm test                # chạy tất cả
npx playwright test --ui  # XEM chạy từng bước ★
npx playwright show-report # bảng pass/fail
npx playwright show-trace # "ghi hình" khi đỏ

★ Người mới luôn bắt đầu bằng --ui — thấy test chạy như xem video.

2Một test trông như thế nào

test('user đăng nhập được', async ({ page }) => {
  await page.goto('/login');           // tới trang
  await page.getByLabel('Email').fill('a@x.dev');
  await page.getByRole('button',{name:'Sign in'}).click();
  await expect(page.getByText('Welcome'))
        .toBeVisible();                // kết quả mong đợi
});

Đọc như một câu: tới login → nhập → bấm → "Welcome" phải hiện. Đúng 3 phần như test case thủ công: chuẩn bị → thao tác → kiểm tra.

★ Thói quen vàng

Luôn XEM TEST FAIL trước khi tin nó.

  1. Chạy → xanh ✅
  2. Cố tình sửa cho sai → chạy lại → phải đỏ ❌
  3. Sửa lại → xanh ✅

Test không thể đỏ = không kiểm gì cả.

3Vòng lặp mỗi ngày với Claude

  1. Xin 1 test nhỏ, bắt đầu luồng quan trọng nhất.
  2. Đọc test. Khó hiểu → "giải thích dòng X".
  3. "Làm test này fail để chứng minh nó kiểm thật."
  4. Thêm case lỗi (thế mạnh của bạn).
  5. Chấm rồi mới nhận.

4Chấm test Claude viết

  • Có test đúng cái mình muốn?
  • Có bắt được bug thật? (làm nó đỏ thử)
  • Kiểm điều người dùng thấy, không phải chi tiết bên trong?
  • Có case lỗi / biên chưa?
  • Không có waitForTimeout, chạy độc lập?
  • Tên rõ, hiểu được sau 6 tháng?

Nghi "xanh giả": assert quá yếu, mock hết, try/catch nuốt lỗi, không có expect.
